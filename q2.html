In Amazon's expansive distribution network, there are n unique inventory processes, each supported by robots[i] robots. These processes are competitive, resulting in clash events every minute between two randomly chosen processes.

 

During a clash:

  • The process with more robots overtakes the one with fewer robots, gaining its robots.
  • If both processes have the same number of robots, one process randomly wins and gains the other's robots.

 

This process repeats until only one inventory process remains.

 

The task is to identify which inventory processes have a possibility of surviving until the end in at least one scenario. Return the indices of these processes (in ascending order, 1-indexed) that could potentially emerge as the last surviving process.

 

Note: The indices of of the inventory processes that needs to be returned are 1-based indexed.

 

Example

n = 5

robots = [1, 6, 2, 7, 2]

 

Assuming 1-based indexing of the robots array:

 

The inventory process at index 2 may survive after the following clashes (where robots[i] = 0, indicates that it's robots have been acquired by some other robot):

  • The 2nd process acquires the 1st process, gaining its robots and increasing its total to 6 + 1 = 7. Hence, robots = [0, 7, 2, 7, 2].
  • Subsequently, it eliminates the 3rd process, raising its total count of robots to 7 + 2 = 9. Now, robots = [0, 9, 0, 7, 2].
  • Next, it defeats the 4th process, enhancing its total count of robots to 9 + 7 = 16. Now, robots = [0, 16, 0, 0, 2].
  • Finally, it conquers the 5th process, increasing its total robots to 16 + 2 = 18. As a result, robots = [0, 18, 0, 0, 0].

 

The inventory process at index 4 can also endure after the following clashes:

  • Initially, the 4th process acquires the 1st process, inheriting its robots and taking its total number of robots to 7 + 1 = 8. Hence, robots = [0, 6, 2, 8, 2].
  • Then it surpasses the 3rd process, with its total number of robots ascending to 8 + 2 = 10. Consequently, robots = [0, 6, 0, 10, 2].
  • Now it overcomes the 2nd process and the total number of robots becomes 10 + 6 = 16. Therefore, robots = [0, 0, 0, 16, 2].
  • Finally, it overcomes the 5th process and its total robot count strengthens to 16 + 2 = 18. As a result, robots = [0, 0, 0, 18, 0].

Hence, these two inventory processes may continue to operate following the clashes. The other processes will be terminated at some point under all possible scenarios.

 

Thus, the answer is [2, 4].

 

Function Description

Complete the function fetchProcessIndices in the editor below.

 

fetchProcessIndices has the following parameter(s):

    int robots[n]: an array of integers depicting the number of robots each inventory process controls.

 

Returns

    int[]: the identifiers of inventory processes in ascending order that could endure, post clashes in at least one probable scenario.

 

Constraints

  • 1 ≤ n ≤ 2 * 105 
  • 1 ≤ robots[i] ≤ 109

 

Input Format for Custom Testing

The first line contains an integer n, the size of the array robots.

Each of the next n lines contains an integer robots[i].

  • The 3rd process overpowers the 4th process and gaining its robots, bringing its total to 5 + 2 = 7. So, robots = [2, 15, 7, 0, 10].

import java.util.*; class Result { public static List fetchProcessIndices(List robots) { int n = robots.size(); Pair[] processes = new Pair[n]; // Pair robot counts with their original 1-based indices for (int i = 0; i < n; i++) { processes[i] = new Pair(robots.get(i), i + 1); } // Sort by ascending count Arrays.sort(processes, (a, b) -> Long.compare(a.count, b.count)); long prefixSum = 0; int start = 0; // Find the start of the final subarray for (int i = 0; i < n - 1; i++) { prefixSum += processes[i].count; // If sum of smaller processes is still less than next process's count, reset if (prefixSum < processes[i + 1].count) { start = i + 1; } } // All from 'start' to the end can survive in at least one scenario List result = new ArrayList<>(); for (int i = start; i < n; i++) { result.add(processes[i].index); } // Return indices in ascending order Collections.sort(result); return result; } // Helper class to hold the count and original index static class Pair { long count; int index; Pair(long c, int i) { count = c; index = i; } } }
